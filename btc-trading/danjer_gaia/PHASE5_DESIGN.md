## 3. 主要コンポーネント

### 3.1 OKX 統合 `exchange/okx_client.py`

Phase 4 で接続検証まで完了済。 Phase 5.1 で本格 Live投入。

```
exchange/
├── okx_client.py          # Phase 5新規 (Phase 4で接続検証済)
├── exchange_router.py     # Bybit/HL/OKX 自動切替 (障害退避)
└── tests/
```

**3取引所障害退避ロジック**:
- WebSocket heartbeat で各取引所の状態監視
- 1取引所が3分以上応答停止 → 他取引所へポジ移管
- Bybit > Hyperliquid > OKX の優先順 (流動性順)

### 3.2 完全自律 Demotion `auto_demotion/`

3者会議 Round 10 合意: 「悪い個体は ミリ秒で paper降格、 昇格は段階的 (非対称設計)」

```
auto_demotion/
├── demotion_engine.py     # 即時降格判定
├── promotion_engine.py    # 段階的昇格 (paper→Shadow→Stage 0→Stage 1→Stage 2)
├── thresholds.py          # 降格・昇格の閾値
└── tests/
```

**非対称設計**:
- **降格**: 1件のロスカット / SL未設置 / Order Gate違反 / DD超過 → **即時paper降格** (ミリ秒)
- **昇格**: paper 30日 + 事故0 + Trade-EHRプラス + 監督負荷低 → Shadow 14日 → Stage 0 7日 → Stage 1 14日 → Stage 2 (本格Live)

### 3.3 自己資本複利再投資 `compound_reinvest/`

Phase 5.3 で導入。 「Shujiさん の生活費を抜いて残りを複利再投資」 ロジック。

```
compound_reinvest/
├── monthly_allocator.py   # 月次の利益から生活費抜く
├── reinvest_scheduler.py  # 残りを段階的に資金注入
└── tests/
```

**ルール案** (3者会議で確定):
- 月利益の 50% は Shuji 生活費 (出金)
- 残り 50% は AI 運用資金に追加
- 累計 Equity が $50,000 を超えたら、 月次出金比率を 30:70 (生活30%、 再投資70%) に変更
- $100,000 超えたら 20:80

### 3.4 合法ヘッジファンド化

**日本居住者の合法的個人運用** (ファンド化禁止、 R6 法規制リスク対策):

- **個人口座**のみ (法人化しない)
- **税務**: 雑所得 (年20-55% 累進)、 確定申告で経費計上 (Cloud Run/Gemini API代等)
- **報告**: 取引所からの年次取引履歴を会計ソフトに連携 (年1回)
- **規模上限**: 個人運用なので「ファンド募集禁止」、 他人資金を預からない

詳細は CPA (公認会計士) と相談、 Phase 5.3 着手前に税務スキーム確定。
