## 評価指標 (2層 + 臆病チェック、 合意通り)

### 🛡 守り層 (退場しない、 = 勝ち続けるための土台)

| 指標 | 目標 | 不合格判定 |
|---|---|---|
| 危ない場面で「やる」 と返した | **0件 (必須)** | 1件以上 = 即修正対象 |
| 強制ストップ7条件のバグ | **0件 (必須)** | バグあり = ルール修正 |
| 同フレーズ無限ループ | **0/N件** | あり = 学習設定見直し |
| 内容崩壊 (9連続/中国語片/空) | **0/N件** | あり = ハーネス再修正 |

### ⚔ 攻め層 (とことん上手くなる、 = ゴールそのもの)

| 指標 | 目標 | 評価方法 |
|---|---|---|
| 良い局面を選べた率 | **継続向上** | 「攻めるべき場面で攻めた」 vs 「攻めるべき場面で見送った」 |
| **仮想損益 1h後** ★合意追記 | **プラス収束** | 信号時点 → 1時間後の価格差 |
| **仮想損益 4h後** ★合意追記 | **プラス収束** | 信号時点 → 4時間後の価格差 |
| **仮想損益 24h後** ★合意追記 | **プラス収束** | 信号時点 → 24時間後の価格差 |
| **最大上昇幅 (24h以内)** ★合意追記 | 大きいほど良い | 信号方向の最大利益 (取り逃した利益も把握) |
| **最大逆行幅 (drawdown、 24h以内)** ★合意追記 | 小さいほど良い | 信号と逆方向の最大損失 (RR管理用) |
| 勝率 | **継続向上** | 勝ち判定 / 全エントリー判定 |
| 平均RR (リスクリワード) | **1:1.5以上** | 想定利幅 / 想定損切幅 |

### 😨 臆病チェック (合意の追加要素、 「何もしないAIにならないか」)

| 指標 | 警告ライン | 対処 |
|---|---|---|
| no_trade率 | **95%超で警告** | 学習データに攻め例を足す、 安全弁比率を下げる |
| 強制ストップ発火率 | **70%超で警告** | 強制ストップ7条件の見直し (過剰反応) |
| 平均応答長 | **30文字未満で警告** | 学習設定見直し (簡素化しすぎ) |

## 強制ストップ7条件 (合意ルール、 LoRAと独立)

(前設計書から継承、 詳細は `stop_rules.py` で実装)

1. 損切り不明 (response に「損切」 も「背」 もない)
2. 損切り近すぎ (SL距離 < ATR×0.5)
3. 根拠なし急騰 (rally regimeで OI急増なし)
4. 根拠不足 (材料2点未満 or 汎用キーワードのみ)
5. 薄い逆張り (crashで反発買い、 trend逆方向ショート等)
6. 板薄/急変 (Volume < 平均30% or ATR > 平均×3)
7. モデルが曖昧 (response に「不明/曖昧/分からない」 含む、 or 20文字未満)

## 本番発注遮断の二重防御 (合意の必須項目)

設定一つで本物の注文が出る事故を絶対防ぐ:

1. **ccxt (取引所API library) を pip install しない**
2. **config.yml の paper_mode=true を起動時 assert**
3. **取引所のAPIキー環境変数 (BYBIT_API_KEY等) を一切読まない**
4. **tests/test_no_order.py で「import ccxt されない」 を自動テスト**
5. **safety_check.py で5項目チェック → 「🟢 SAFE」 表示まで起動拒否**

```bash
$ python3 safety_check.py
✅ ccxt not installed
✅ config paper_mode=true
✅ no API keys in env
✅ regime_detector only reads OHLCV
✅ inference only reads/writes local files
🟢 SAFE: this environment cannot place real orders
```

## 育成サイクル 期間

- **1サイクル**: 3週間 × 50〜100件判定 (合意通り)
- **継続**: N サイクル (終了条件は明確には決めない、 合格基準を上げる難易度カリキュラム)
- 各サイクル末で:
  - Shujiさん判定 → 教師データ化
  - 再学習 (v(N+1)) → Kaggle 6h
  - 強制ストップ7条件の見直し
  - 次サイクルの「合格基準」 を上げる (例: vN は no_trade率80%許容、 v(N+1) は no_trade率50%以下 等)

### ★合意追記: 実弾¥10万 移行条件 (必須)

実弾¥10万の判断は **以下2条件 全てクリア** してから:

1. **最低3サイクル (約2ヶ月) の育成を完了** している
2. **臆病警告 (no_trade率95%超 / 強制ストップ70%超 / 応答長30文字未満) がゼロ** になっている

ただし、 これは「条件をクリアしたら自動でGO」 ではなく、 **Shujiさん自身が結果を見て最終判断**。 AIの多数決外 (合意明記)。
