## 2. サイクル理論 -- 最重要手法

danjerが **最も頻繁に使用し、最も信頼している手法**。ツイート250件以上がサイクル関連。
「日足サイクルがワイのメイン取引手法」(2025-08-01) と明言。

### 2.1 サイクルの階層構造

| サイクル | 期間 | 使い方 |
|---------|------|--------|
| **1500日サイクル** | 約4年 | 半減期サイクル。2026/2頃まで上昇見込み |
| **500日サイクル** | 300日+200日 | 超長期のメジャーサイクル。起点で長期ロング |
| **300日サイクル** | 約10ヶ月 | 中期サイクル。3日足で確認 |
| **90日サイクル** | 約3ヶ月 | **月足レベルの方向判断**。陰線3ヶ月続くと4ヶ月目は陽線率高い |
| **30日サイクル** | 約1ヶ月 | **日足レベルのメイントレード**。月初に安値が来るパターン多い |
| **4時間サイクル** | 60-80本 (10-13日) | **最頻使用。短期トレードの核心** |

### 2.2 4時間サイクルの具体的ルール

**基本定義**: 安値 → 安値 をローソク足の本数で計測

**本数の目安**:
- **上昇トレンド時**: 70-90本 (ゆっくり上がるので長い)
- **レンジ/通常時**: 60-80本 (80%の確率で安値)
- **下降トレンド時**: 50-65本 (展開が早いので短い)

**具体的な運用**:
```
1. 前回安値の本数を記録
2. 直近3-5サイクルの平均本数を算出
3. 現在の本数が目安ゾーン(60-80本)に入ったら「拾いZONE」
4. ライトトランスレーション(高値が右側) → 上昇トレンド示唆
5. レフトトランスレーション(高値が左側) → 下降トレンド示唆
```

**danjerの実際の予測と結果** (抽出サンプル):
- 5/12-14底予想 → 5/11確定(56本) -- 的中
- 5/23 AM5:00安値予想(73本目) → 的中
- 8/28 AM5:00サイクル終了(72本) → 的中
- 9/5近辺安値予想 → 9/6安値 -- 的中
- 9/17-19安値予想(61本目) → 的中

**重要な注意点**:
- 「サイクルは横軸の考察。縦軸は適当にLINE引いてるだけ」(2024-07-13)
- 「サイクルはレンジが課題」(2025-05-20) -- レンジ相場では精度が落ちる
- 「理論は80%しか当たらない」(2024-12-06)

### 2.3 30日サイクル

**BTC日足で30本前後で安値 → 安値**

**運用ルール**:

- 30日サイクルの1つ目は高値が右側(ライトトランスレーション)の可能性が高い

### 2.4 90日サイクル

**3ヶ月ごとの安値パターン**:
```
2024/8/5 → 2024/11/4 → 2025/2/13 → 2025/4/7 → 2025/6/22 → 2025/9月初旬?
```

**ルール**:

- 90日サイクル内の最初の30日サイクルは上昇しやすい
- 「陰線3ヶ月続くと4ヶ月目は陽線率高い」

### 2.5 トランスレーション判定

| 判定 | 条件 | 意味 |
|------|------|------|
| ライトトランスレーション(RT) | 高値がサイクル中心より右(後半) | **上昇トレンド** |
| レフトトランスレーション(LT) | 高値がサイクル中心より左(前半) | **下降トレンド** |

**具体的運用**:
- RT確定 → 安値でロング追加、次のサイクルも上の可能性
- LT確定 → 早めに利確、ショートも検討
- 前回サイクルの始値(安値)割れ → サイクル崩壊、ロング手仕舞い

## 3. エントリー条件

### 3.1 ロングエントリー

#### A. サイクル底ロング (最頻出)
```
条件:
1. 4Hサイクルが60-80本目ゾーンに到達
2. 直近安値付近 or Wボトム形成
3. OI精算(ロングのロスカット)が入った直後
4. フィボナッチ半値 or サポートライン付近

エントリー:
- 「うすーく買い下がり」(小ロット分割)
- 指値を安値付近にばら撒く
- AM5:00(NY引け)の反転を狙う

利確:
- 1本目: 日足始値で利確
- 2本目: 建値ストップして伸ばす
```

#### B. MA収束→拡散ロング (鉄板パターン)
```
条件:
1. 4H足MAに対して1H足MAが収束
2. 1H MAが4H MAを下から上にクロス(拡散開始)
3. 近くにレジスタンスがない
4. RR 1:2以上

エントリー:
- クロス確認後に成行
- ストップ: 直近安値
```

#### C. レジサポ転換ロング
```
条件:
1. レジスタンスを上抜け
2. ローソク実体で確定(髭だけではNG)
3. 押し戻しで旧レジスタンス(新サポート)をテスト
4. サポート確認後にロング

ストップ: 新サポート割れ
```

#### D. 精算後リバ取りロング
```
条件:
1. 急落でロングのロスカット大量発生(Bybit精算確認)
2. OI急減
3. 直近安値 or サポートライン到達
4. 1分-5分足で反転の兆候

エントリー: 精算確認後、即ロング
利確: 短期(謙虚利確)
```

### 3.2 ショートエントリー

#### A. サイクル天井ショート
```
条件:
1. 4Hサイクルの高値更新が限定的
2. 3段上げ完了後
3. FR過熱(+方向に偏り)
4. OI増 + 価格横ばい = つっこみロンガー増加

エントリー: 高値更新失敗 or ピンバー確認後
```

#### B. MACD/ダイバージェンスショート
```
条件:
1. 8H足 or 日足でMACDダイバージェンス
  (価格: 高値切り上げ、MACD: 高値切り下げ)
2. ヒドゥンダイバージェンス + DCでショート
3. ウェッジ上限 or チャネル上限到達

ストップ: 直近高値
利確: ウォルフ波動ライン or 半値
```

#### C. 戻り売りショート
```
条件:
1. 下落トレンド確認(MTFで上位足が下)
2. 4H SMA100 or SMA200にタッチ
3. グランビルの法則5(新規売り)or 6(ロールリバーサル)
4. ローソク実体でMA下で確定

ストップ: MA上
```

### 3.3 エントリーしない条件

- **レンジの中央**: 「レンジの中で無理にポジらない」
- **FOMC/CPI等の重要指標前**: 「FOMCまでにノーポジにしたい」
- **金曜日**: 「金曜は触らない」(曜日アノマリー)
- **ボラが死んでいる時**: 「ボラが死んでるから一撃での復活とか無理」
- **ツッコミ(急騰中に飛び乗り)**: 「ツッコミSはせずに静観」「ツッコミロングはしない」

## 5. OI/FR/精算の判断基準

danjerのOI分析は **非常に体系的**。2022年に集中的にOIとFRの関係を分析し、それ以降一貫して使用。

### 5.1 OI + FR + 価格の組み合わせ判定表

| 価格 | OI | FR | 判定 | アクション |
|------|-----|-----|------|-----------|
| 下落 | 増加 | (-) | ショートが積まれた | ショートカバーでリバの可能性 |
| 下落 | 増加 | (+) | ロングがナンピン中 | さらに下落の可能性、ショート継続 |
| 下落 | 減少 | (-) | ロングの利確 | 売り圧減少、リバの兆候 |
| 上昇 | 増加 | (-) | ナンピンショート | ショートカバーでさらに上の可能性 |
| 上昇 | 増加 | (+) | 新規ロング積み | トレンド継続だが過熱注意 |
| 上昇 | 減少 | 改善 | ショートの利確 | 持続力なし、一過性のリバ |
| 横ばい | 増加 | フラット | 煮詰まり | ブレイク間近、方向は不明 |

### 5.2 精算(Liquidation)の使い方

**基本原則**: 「OI積まれたら精算が入って短期の転換になる」

```
ロング精算 → 短期の買い場(売り圧消化)
ショート精算 → 短期の売り場(買い圧消化)
```

**具体的トレード**:
1. CoinGlassで精算ヒートマップ確認
2. 大量精算が入った直後にカウンター
3. 「精算確認してぴょんと跳ねたとこで短期ショート」
4. 「ロスカット精算特大発生 → 62割れで拾う」

### 5.3 FR(ファンディングレート)の使い方

- **FR (+) 過熱**: ロンガー過多 → 調整下落警戒
- **FR (-) 過熱**: ショーター過多 → ショートカバーで上昇の可能性
- **FR フラット**: 需給ニュートラル → チャートパターンで判断

**アルトのFR過熱は特に危険**: 「FRがアルト中心にデリバ過熱ぎみ → 危険信号」

### 6.1 フラクタルの使い方

**「チャートはフラクタル。過去を繰り返す事が多い。同じ事を繰り返し、利益を上げてます」** (2024-05-20)

```
手順:
1. 現在のチャートパターンに似た過去のチャートを探す
2. 過去のローソク足をコピーして現在のチャートに重ねる

```

```

```

```
- 上昇時はゆっくり、下落時は早い(時間軸の非対称性)
```

#### ウォルフ波動
```
- 5波終了後の利確目標として使用
- 1→4ラインで利確ターゲットを設定
- 下降チャネルのウォルフ波動も活用
```

```
90日サイクル + エリオット5波動:
```

## 8. 移動平均線・MACD・一目均衡表

### 8.1 移動平均線(SMA/EMA)

**使用MA**:
- **SMA100**: 最も重視。「4H SMA100は最強」「ETH 4H SMA100は無敵」
- **SMA200**: 長期トレンド判定。「週足SMA200到来 → 大底候補」
- **GMMA**: トレンドの可視化。「GMMAが最強」「乖離を用いた逆張りも」

**具体的ルール**:
```
グランビルの法則:
1番(買い): SMA100がフラットor上向きに変わり、ローソクが上抜け
3番(買い増し): SMA100付近への押し戻しから反発
5番(新規売り): SMA100を下抜けて実体確定
6番(戻り売り): SMA100への戻りから再下落

SMA100-200レンジ:
- 「SMA100-200はレンジになることが多い」
- ボラ縮小後にトレンド発生
- SMA200割れ → トレンド転換
```

### 8.2 MACD

**使用時間足**: 5分, 30分, 1H, 8H, 日足, 週足

**具体的シグナル**:
```
ロング: MACDダイバージェンス(価格安値切下, MACD安値切上) + GC
ショート: ヒドゥンダイバージェンス(価格高値切下, MACD高値切上) + DC

5分足MACD:
- 「5分足 + MACD(ヒドゥンダイバGC) = ロング!!!」
- 節目のブレイクアウト確認に使用

週足MACD:
- 「週足MACDダイバ + GC → 大底」
- 「前回はMACDのGCはほぼ頂点」(反例もある)
```

### 8.3 一目均衡表(雲)

**使用時間足**: 2H, 4H, 8H, 日足

**具体的ルール**:
```
雲がサポート:
- 「雲を登ってくイメージで上」
- 「雲中INしたらレンジ → 下ブレイク」
- 「4H雲に入ったら一回折れる可能性高い」

雲のねじれ(転換):
- 「ねじれ = 転換点。前回はこのねじれで上昇」
- 雲のねじれ日程を事前に確認、サイクルと組み合わせ

雲の厚さ:
- 「雲が薄くなったところを狙った上げ」
- 薄い雲 → ブレイクしやすい
- 厚い雲 → 強いサポート/レジスタンス
```

### 8.4 ボリンジャーバンド

```python
def should_long(data):
    score = 0

    # 1. サイクル底判定 (最重要、weight=40%)
    if data.cycle_4h_bars >= 60 and data.cycle_4h_bars <= 90:
        score += 40  # サイクル底ゾーン

    # 2. OI/精算シグナル (weight=20%)
    if data.oi_change < 0 and data.long_liquidation_spike:
        score += 20  # ロング精算入り = 売り圧消化

    # 3. フィボ/サポート (weight=15%)
    if data.price_near_fib_50 or data.price_at_support:
        score += 15

    # 4. MA (weight=10%)
    if data.price_above_sma100_4h and data.sma_converging:
        score += 10

    # 5. アノマリー (weight=10%)
    if data.is_monday or data.is_saturday_am5:
        score += 10
    if data.is_thursday or data.is_friday:
        score -= 10  # 木金はロング非推奨

    # 6. 時刻 (weight=5%)
    if data.hour_jst == 5:  # AM5:00
        score += 5

    return score >= 60  # 閾値60%以上でエントリー
```
