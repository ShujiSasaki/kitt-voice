## 1. 手法の全体像

ronpochi(ロンポチ)のトレード手法は、**フラクタル分析**を中心としたユニークなアプローチ。
「過去のチャートパターンが現在に繰り返される」という前提に立ち、ライントレード(チャネル・フィボナッチ・水平線)と組み合わせてエントリーポイントを特定する。

### 手法の核心を本人が定義した言葉

> 「基本はライントレードそしてフィボ。それに直近の動きのフラクタル、大きな流れのフラクタルの分析からポイントを絞ってエントリーです」(2019-12-19)

> 「フラクタルのみでエントリーはしてないですよ、流石にw」(2019-12-19)

> 「フラクタルを単純にトレースした動きで見てない。フラクタルを現在の動きに落とし込んだ、自分なりの分析で見てます」(2020-05-30)

> 「フラクタルで自分はやってるけど、何故フラクタルが通用するかってのは、過去チャートは結局エリオット、フィボ等で意識して動いてる集大成みたいなものなので、似た動きはするのは必然」(2020-06-21)

## 2. エントリー条件の完全解明

### 2.1 分析の階層構造 (大→小)

ronpochiはチャートを3つの時間スケールで分析する:

| スケール | 内容 | 参照先 |
|---------|------|--------|
| 大局フラクタル | 2017-2018年バブル崩壊チャートとの類似 | 月足/週足/日足 |
| 中期フラクタル | 直近数週間~数ヶ月の動きの繰り返し | 日足/4時間足 |
| 短期フラクタル | 直近数日の動きの繰り返し | 1時間足/15分足 |

**本人の言葉**:
> 「フラクタル研究家のアタクシは、大きな流れはバブル崩壊、そこそこ動きはちょっと前の動き、直近の動きは近くに似てる動きをしてると思ってます」(2019-10-08)

### 2.2 チャート確認の順序 (固定ルール)

> 「チャート見る時は必ず大きな足から毎回見てます。月足→週足→3日足→日足→4時間足→1時間足→15分足。短期足から見ると、どうしても短期の動きが印象に残るので。木を見て森を見ずですね」(2020-01-24)

### 2.3 ロングのエントリー条件

**具体的な条件の組み合わせ**:

1. **チャネル下限タッチ**
   - チャネルの下限に到達、または半値ラインで反発
   - 「チャネルの半値とか半値の半値とかは良く反発するなーってイメージ」(2019-10-14)
   - 「チャネルの半値の半値の半値は反発するイメージ」(2019-10-20)

2. **フィボナッチリトレースメント反発**
   - 主要レベル: **0.382**, **0.5**, **0.618**
   - 「フィボの0.5ラインとレジスタンスのラインにちょうど重なるので抜くのはきつい」(2019-10-09)
   - 「0.618ラインで反発して耐えてますね。8619でロング入れてみました」(2019-11-14)
   - フィボ0.618 + チャネル下限 + 水平線が重なるポイントが最強

3. **フラクタルパターンの一致**
   - 過去のチャートと現在の動きが一致する場面でエントリー
   - 時間軸の倍率を計算(例:「約2倍のスピードで動いてる」「約3から4倍のスピード」)
   - バブル崩壊チャート(2018年)の主要反転ポイントに該当する位置

4. **下ヒゲ確認(重要)**
   - 「下ヒゲボーボー確認してからで良いかと」(2019-10-11)
   - ガラ(暴落)後の反発を確認してからエントリー。突っ込みロングはNG

5. **ダブルボトム / 逆三尊形成**
   - ただし「右肩下がりの逆三尊は成立は騙しになるか、成立しない印象」(2020-03-01)

**ロングしない条件**:
- 「突っ込みロングは危険」(2019-10-29)
- 上げトレンドの途中で追いかけない
- 反発が確認できていない状態

### 2.4 ショートのエントリー条件

1. **チャネル上限タッチ**
   - チャネル上限到達でショート。予測チャネルも含む
   - 「このは素直に予想チャネル上限でショート、抜けたら損切りで、えーかな？」(2020-03-23)

2. **ダブルトップ形成**
   - 最頻出パターン。高値を2回試して失敗→ショート
   - 「三尊否定→ダブルトップで落ちるってイメージ」(2020-02-01)
   - 「高値切り下げのダブルトップマン参上！！」(2020-03-11)

3. **フィボナッチの0.236 / 0.382からの反落**
   - リバウンドがフィボの浅いレベルで止まった場合

4. **フラクタルの天井パターン一致**
   - バブル崩壊の「3つ目の山」パターンが最重要の大局ショートサイン

5. **売り上がり戦略**
   - 上昇中に分割でショートを入れていく
   - 「朝上げからの下げあると思うので、チャネル上限あたり、8400から8500でショートばらまいときます」(2019-10-02)

6. **大きな上ヒゲの確認**
   - 「時間足レベルでの大きな上髭がまだない！！(かなり私の中で大事)」(2020-04-19)

### 2.7 フラクタル分析の具体的手法

1. **参照チャートの選定**: 現在の動きと最も似た過去の動きを見つける
2. **時間軸の倍率計算**: 過去の動きが何倍のスピードで再現されているかを計算
   - 例: 「約2倍のスピードで動いてる」→ 過去10日かかった動きが5日で進行
   - 例: 「約3から4倍のスピードで動いてる」
3. **ターゲット算出**: 過去の高値・安値をチャネル・フィボに当てはめて目標価格を算出
4. **時間軸予測**: 倍率から「カチ上げは28日あたりか？」のように日付も予測
5. **乖離チェック**: 予想値と実際の値のズレを常にモニタリング(「予想より100ドルずれる」)

**フラクタル分析の強み(本人認識)**:
> 「大きな動きの高値安値は、これまでの分析から勝率は高いんです」(2020-01-31)

**フラクタル分析の弱み(本人認識)**:
> 「細かい動き、例えば途中の反発ポイントとか、ダブルトップで落ちるとかは、なかなかフラクタル的にははまらない」(2020-01-31)

## 3. イグジット条件の完全解明

### 3.1 利確の基準

| パターン | 条件 | 具体例 |
|---------|------|--------|
| **ターゲット到達** | フラクタル/チャネルから算出した目標価格に到達 | 「ターゲットは8400ドルあたり」 |
| **チャネル上限/下限到達** | 反対側のチャネル限界に到達 | 「チャネル上限8130あたりで叩かれると思います」 |
| **フィボレベル到達** | 目標のフィボレベルに到達 | 「フィボ0.618あたり」 |
| **ストップ利確引き上げ** | 含み益が出たらストップを引き上げて放置 | 「ストップ利確入れて勝確です」 |
| **動きが予想と違う** | イメージと違う動きの場合は早期利確 | 「思ったよりも弱いのが気になります」 |
| **時間軸の消費** | フラクタルの予想時間を過ぎても動かない場合 | 「5日までに8600あたりにこなければ」 |
| **分割利確** | 1/3利確→残りは建値ストップ | 「三分の1は8300で利確済」 |

### 3.2 損切りの基準

| パターン | 条件 | 具体例 |
|---------|------|--------|
| **直近高値/安値更新** | チャネル/水平線をブレイク | 「損切りは直近の高値8532なんで、8540あたりで」 |
| **チャネルブレイク** | チャネル上限(ショート)/下限(ロング)の明確なブレイク | 「抜けたら損切りで」 |
| **安値更新(ロング)** | 前回安値を割ったら即撤退 | 「安値更新で損切り予定」 |
| **高値更新(ショート)** | 高値をブレイクしたら撤退 | 「高値あたりにストップいれて放置中」 |
| **同値撤退** | 方向感がなくなったら同値で逃げる | 「同値撤退を入れておきます」 |
| **建値ストップ** | 含み益が出たら建値にストップ→最悪でもイーブン | 「ロングは建値でストップ放置」 |
| **予想との乖離** | フラクタルの予想と大きくズレた場合 | 「予想のイメージと違う動きなんで、一旦損切りするかもです」 |

**損切り幅の実績データ**:
- 典型的な損切り幅: **20~50ドル** (当時のBTC $7,000-$10,000帯)
- パーセンテージ換算: 約 **0.3~0.7%**
- 分割エントリーの場合はより広い幅を許容(100-200ドル = 1-3%)

### 3.3 損切り後のルール

> 「損切りした日はポジらないので、また夜に考えたいと思います」(2019-10-06)
> 「自分ルールで損切りしたらノーポジって縛りがある」(2019-10-10)
> 「損切りした時は負けを取り戻そうとして冷静な判断が出来ず、負けが増える事あります」(2019-10-10)

**これは明確なルール**: 損切り後はその日ノーポジ。翌日以降に再分析してからエントリー。

### 3.4 トレイリングストップ

明示的な「トレイリングストップ」の言及はないが、実質的に同等のことをやっている:
- 「ストップ利確引き上げながらついていくつもり」(2020-07-02)
- 「半分は利確ストップかかってます。残り半分建値でストップ」(2020-07-05)

## 4. リスク管理

### 4.1 レバレッジ

ronpochiは**具体的なレバレッジ倍率をほぼ言及していない**。BitMEX(メックス)とBybitを使用しており、ハイレバが可能な取引所だが、倍率の明示はない。

間接的な情報:
- 「維持率と相談してください」(多数) → ポジションが大きく維持率を気にしている
- 「ギリギリ維持率100切るかもね」(2020-03-08) → かなり攻めたレバの時もある
- 「資金追加した」 → 維持率がギリギリの時に追加証拠金を入れている

**推定レバレッジ**: 維持率を常に気にしていることから、**5x-25x程度**と推定。100倍フルレバではないが、低レバでもない。

### 4.2 ポジションサイズ

- 基本: 「コントロールできるエントリーが10としたら5で基本エントリーしてます」(2020-03-05)
  → **最大ポジションの50%**が1トレードの上限
- 分割エントリー: その50%をさらに**5分割**
  → 1回のエントリーは最大ポジションの**10%**
- 自信度で調整: 自信がある時は枚数を増やし、ない時は「少しだけ」「軽く」

### 4.3 分割エントリーの具体例

```
初弾: 8166 (10%)
追加1: 8220 (10%)
追加2: 8292 (10%)
追加3: 8314 (10%)
追加4: (動き次第) (10%)
合計: 最大50%
```

- 5分割が全て刺さって予想範囲内なら逆でも持つ
- 予想外の動きなら初弾のみで撤退

### 4.4 最大ポジション数

- **基本1ポジション(ロングかショート)**
- 2020年6月以降: **両建て**(ロング+ショート同時保有)を導入
  - メインポジション + ヘッジポジションの2つ
  - 「リスク少なくして、上でも下でも大きい動きを取るため」
  - 「禁断の両建て」と本人も認識

### 4.5 資金管理の格言

- 「生き残れば、チャンスなんていくらでもあります！」(2019-10-04)
- 「損切り設定までが遠足です！」(繰り返し使用)
- 「逃げ足は光の速さで」(繰り返し使用)
- 「頭と尻尾はくれてやれ」

## 5. 相場環境認識

### 5.1 ブル/ベアの判断基準

| 判断 | 条件 |
|------|------|
| **上目線** | チャネル下限で反発 + フラクタルが上昇パターンに一致 + 安値切り上げ |
| **下目線** | チャネル上限で反落 + フラクタルが下降パターンに一致 + 高値切り下げ |
| **フラット** | 「フラクタルは下、データは上」のように相反する場合 |

> 「大局目線: 上目線フラクタル→微妙に崩れ気味。インジ等→上じゃね？データその他→下じゃね？ローソク足→ソーサトップじゃね？ うん、フラット目線だね」(2020-06-30)

### 5.2 トレードしない条件

1. **方向感がない時**: 「わからない時はノーポジ」(最頻出ルール)
2. **損切り直後**: 「損切りした日はポジらない」(固定ルール)
3. **チャートに張り付けない時**: 「仕事中なのでノーポジ」(リーマン)
4. **フラクタルがハマらない時**: 「フラクタル分析にハマらないので今は予想しにくい」
5. **ファンダメンタルが入り乱れる時**: 半減期前、テザー問題等
6. **先物SQ前後**: 「先物SQもあるし6月までポジション取りにくい」
7. **中途半端な価格帯**: 「中途半端な位置のポジはやめた方がえーです」
8. **暴落直後**: 「なんで、この動きで新規エントリーする人いるのかな？バブル崩壊経験者の自分は怖くて触れない」

### 5.3 曜日・時間帯パターン

- 「木曜日はだいたい下がると言われてますね」(2019-10-02)
- 「金曜日の魔術師。下げた分上げる魔術師」(2020-03-06)
- 朝上げ→日中に戻すパターンを意識

## 6. 実際のトレード事例の分類

### 6.1 最頻出セットアップ (成功パターン)

#### パターン1: チャネル上限ショート (最多)
```
条件: チャネル上限到達 + フラクタルの天井パターン
エントリー: チャネル上限付近で分割ショート
損切り: チャネル上限の少し上(20-50ドル)
利確: チャネル半値 or チャネル下限
勝率: 高い(本人談「大きな動きの高値安値は勝率高い」)
```

実例:
- 2019-10-02: 8400-8500ショート → 成功
- 2020-01-12: チャネル上限で分割ショート → 成功
- 2020-01-23: 8588ショート → 8300で1/3利確 → 成功

#### パターン2: フィボ + チャネル下限ロング
```
条件: フィボ0.382/0.5/0.618 + チャネル下限が重なるポイント
エントリー: 重複ポイント付近で分割ロング
損切り: チャネル下限の少し下
利確: チャネル半値 or チャネル上限
```

実例:
- 2019-09-30: 7760ロング → 利確成功
- 2019-11-14: 0.618ライン8619ロング → 反発で利確
- 2020-01-06: 8100-8200ロング → フラクタル通り上昇

#### パターン3: ダブルトップショート
```
条件: 2回目の高値トライで失敗(前回高値に届かない or 僅かに超えて反落)
エントリー: 2回目の高値付近でショート
損切り: 高値の明確な更新
利確: 直近安値 or チャネル下限
```

実例:
- 2019-12-19: ダブルトップ7500でショート5分割 → 成功
- 2020-02-01: 9550付近でダブルトップショート → 成功
- 2020-03-03: 8960ダブルトップショート → 成功

#### パターン4: 両建て + 方向判明後に片方カット (2020年後半)
```
条件: 方向感不明だが大きな動きが予想される場面
エントリー: ロング + ショート両方保有
決済: 方向判明したら逆ポジをストップ利確、順ポジを伸ばす
```

### 6.2 失敗パターン

#### 失敗1: フラクタル不一致で担がれ
- 予想より100ドル上/下にズレてショート/ロングが担がれる
- 「予想より100ドルずれるなぁ」(2019-10-31)
- 対処: 損切り or ナンピン(平均を改善)

#### 失敗2: ヨコヨコで判断できずフライングエントリー
- 方向感がない中でフライングエントリーして損切り
- 「フライングロング7942の自分はちょっと嫌ーな感じ」(2019-10-06)

#### 失敗3: 握力不足(早すぎる利確)
- 「利確早すぎた」「ロング捨てたの失敗だったか」
- ストップが近すぎて小さな押しで切られる

#### 失敗4: 寝落ちによるストップ未設定
- 「ストップ入れないで寝落ちしてました。寝落ちはダメ、絶対ダメ」(2020-01-01)

## 7. 年別の手法変遷

### 2019年9月-12月 (初期)

- **基本スタイル**: フラクタル分析 + チャネル + フィボ
- **参照チャート**: 2017-2018年バブル崩壊チャートがメイン
- **ポジション**: ロング or ショートの片方向のみ
- **取引所**: BitMEX(メックス)メイン
- **特徴**: 先出し(事前予想公開)を積極的に実施。エントリー価格を具体的に公開
- **課題**: 「予想より100ドルずれる」問題が頻発

### 2020年1月-3月 (成長期)

- **半減期との格闘**: ファンダ(半減期上げ期待)とテクニカル(フラクタルの下目線)の矛盾
- **コロナショック(3月)**: 予想以上の暴落速度に翻弄。「動きが早すぎ」
- **重要な学び**: 「半減期無視すれば完全に下目線」→ファンダを過信しない
- **分割エントリー明確化**: 5分割の具体的ルール確立
- **メックスフリーズ戦略**: 「メックスフリーズ=大底反発」を裏技として発見

### 2020年4月-6月 (進化期)

- **ドテン戦略導入**: 利確→ドテン(反対方向エントリー)を積極的に使用
- **USDT建てトレード**: バイビットUSDT建てでリスク分散
- **慎重化**: コロナショック後、「先出しで呟けない」場面が増加

### 2020年7月-9月 (完成期)

- **両建て本格導入**: 「禁断の両建て」を常用化

## 8. 自動売買への実装可能なルール

### 8.1 実装可能なエントリールール

```python
# === ショートエントリー ===
# 条件1: チャネル上限タッチ
if price >= channel_upper * 0.995:
    SHORT entry (分割5回, 各10%)

# 条件2: ダブルトップ検出
if is_double_top(highs, tolerance=0.3%):
    SHORT entry (分割5回)

# 条件3: フィボ0.236からの反落(弱いリバウンド)
if fib_retrace <= 0.236 and candle_pattern == "bearish":
    SHORT entry

# === ロングエントリー ===
# 条件1: チャネル下限タッチ
if price <= channel_lower * 1.005:
    LONG entry (分割5回, 各10%)

# 条件2: フィボ0.618 + チャネル下限の重複
if abs(price - fib_618) < threshold and abs(price - channel_lower) < threshold:
    LONG entry (高確信度, 枚数増)

# 条件3: ダブルボトム検出
if is_double_bottom(lows, tolerance=0.3%):
    LONG entry

# 条件4: 大きな下ヒゲ確認後
if lower_wick / body_size > 2.0 and timeframe >= "1h":
    LONG entry
```

### 8.2 実装可能なイグジットルール

```python
# === 損切り ===
# チャネルブレイク
if position == LONG and price < channel_lower * 0.995:
    CLOSE LONG

if position == SHORT and price > channel_upper * 1.005:
    CLOSE SHORT

# 固定損切り幅: 0.5-1.0%
stop_loss_pct = 0.007  # 0.7%

# === 利確 ===
# チャネル反対側到達
if position == LONG and price >= channel_upper * 0.99:
    CLOSE LONG (or 1/3 partial)

if position == SHORT and price <= channel_lower * 1.01:
    CLOSE SHORT (or 1/3 partial)

# ストップ利確引き上げ
if unrealized_pnl > 0.5%:
    trailing_stop = entry_price  # 建値ストップ
if unrealized_pnl > 1.0%:
    trailing_stop = entry_price + 0.5% * direction
```

### 8.3 実装可能なリスク管理ルール

```python
# 1回のトレード最大リスク: 口座の10% (5分割 x 2% each)
max_position_size = account_balance * 0.50  # 口座の50%

# 損切り後クールダウン
if last_stop_loss_today:

if confidence < 0.6:
```

### 8.4 実装が困難な要素

以下はronpochiの手法の核だが、自動化が極めて難しい:

1. **フラクタル分析**: 「似たチャートパターン」の検出は主観的。パターンマッチングアルゴリズムで近似可能だが精度は落ちる
2. **時間軸の倍率**: 「2倍のスピード」等の判断は視覚的・感覚的
3. **複数ストーリーの同時保持**: 「何個かストーリーを考えてどの動きかな」→ 条件分岐ツリーで近似可能
4. **雰囲気の読み**: 「雰囲気的にはショート持ちの方がまだ担がれてる感」→ センチメント分析で代替
5. **板読み(アーサー板)**: 特定の大口注文の検出→ APIで取得可能だが判断は困難

## 9. トレードスタイルの要約

| 項目 | 内容 |

| **スタイル** | デイよりのスイング(数日~2週間保有) |

| **利確** | チャネル反対側。分割利確(1/3ずつ) |
| **最大の武器** | フラクタル分析(大局の天底は高勝率) |
| **最大の弱点** | 細かい動きの予測精度(フラクタルでは読めない) |
| **メンタルルール** | 損切り後ノーポジ/わからない時はノーポジ/逃げ足は光の速さ |

## 10. danjer手法との比較・統合の可能性

| 項目 | ronpochi | danjer |
|------|---------|--------|
| 核心手法 | フラクタル分析 | サイクル理論 |
| 共通点 | 過去の動きが繰り返される | 一定の時間周期で動く |
| 時間足 | 4H/日足中心 | 4H/日足中心 |
| エントリー | チャネル+フィボ | エリオット+フィボ |
| 独自要素 | 時間軸の倍率計算 | OI/FR/アノマリー |
| 弱点 | オンチェーンデータ不使用 | フラクタル不使用 |

**統合の方向性**:
- ronpochiの「フラクタルパターンマッチング」+ danjerの「サイクル理論による時間予測」は相互補完的
- ronpochiのチャネル/フィボ手法はそのまま自動売買に組み込み可能
- danjerのOI/FR判断をronpochiの環境認識フィルターとして追加可能
